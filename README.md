# Интернет-магазин «Web-Larёk»

«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где
пользователи могут просматривать товары, добавлять их в корзину и оформлять
заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра
деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный
цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter),
которая обеспечивает четкое разделение ответственности между классами слоев
Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и отвечает за связь
представления и данных.

Взаимодействие между классами обеспечивается использованием
событийно-ориентированного подхода. Модели и Представления генерируют события
при изменении данных или взаимодействии пользователя с приложением, а Презентер
обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component

Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут
быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за
отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает
данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля
класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах,
которые будут наследоваться от `Component` будут реализованы сеттеры для полей с
данными, которые будут вызываться в момент вызова `render` и записывать данные в
необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` -
утилитарный метод для модификации DOM-элементов `<img>`

#### Класс Api

Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор
передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для
запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в
параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` -
принимает объект с данными, которые будут переданы в JSON в теле запроса, и
отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По
умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен
заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод
проверяющий ответ сервера на корректность и возвращающий объект с данными
полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter

Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и
подписываться на события, происходящие в системе. Класс используется для связи
слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` - хранит коллекцию подписок на
события. Ключи коллекции - названия событий или регулярное выражение, значения -
коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` -
подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события.
При вызове события в метод передается название события и объект с данными,
который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` -
возвращает функцию, при вызове которой инициализируется требуемое в параметрах
событие с передачей в него данных из второго параметра.

### Типы данных:

#### Основные типы и интерфейсы

`ApiPostMethods` - тип HTTP-методов для POST-запросов: `'POST'`, `'PUT'`, `'DELETE'`

`IApi` - интерфейс API-клиента с методами `get<T>(uri: string): Promise<T>` и `post<T>(uri: string, data: object, method?: ApiPostMethods): Promise<T>`

`TPayment` - тип способа оплаты: `'online'` (онлайн-оплата), `'cash'` (наличные), `''` (не выбран)

`IProduct` - интерфейс товара, содержит поля: `id` (идентификатор), `description` (описание), `image` (URL изображения), `title` (название), `category` (категория), `price` (цена, может быть `null`)

`ICustomer` - интерфейс данных покупателя, содержит поля: `payment` (способ оплаты), `email` (электронная почта), `phone` (номер телефона), `address` (адрес доставки)

`TErrorsValidate` - тип для хранения ошибок валидации, представляет собой объект с ключами-строками и значениями-строками

#### Типы для работы с сервером

`IFetchItemsResponse` - интерфейс ответа сервера при получении списка товаров, содержит поля: `items` (массив товаров `IProduct[]`), `total` (общее количество товаров)

`IOrderData` - интерфейс данных для отправки заказа на сервер, расширяет `ICustomer`, добавляет поля: `items` (массив ID товаров), `total` (общая стоимость заказа)

`IPostOrderResponse` - интерфейс ответа сервера при успешном оформлении заказа, содержит поля: `id` (идентификатор заказа), `total` (общая стоимость заказа)

#### Типы для работы с компонентами

`IHeader` - интерфейс для компонента заголовка, содержит данные для отображения в шапке сайта

`IModal` - интерфейс для модального окна, определяет структуру данных для управления модальным окном

`IEmitDefault<T extends object>` - базовый тип для событий с данными, содержит поля: `data` (данные произвольного типа), `event?` (опциональное событие DOM)

`TEmitProduct` - тип события с данными товара, является псевдонимом для `IEmitDefault<IProduct>`

`TEmitPartialUserData` - тип события с частичными данными покупателя, является псевдонимом для `IEmitDefault<Partial<ICustomer>>`

`TEmitUserData` - тип события с полными данными покупателя, является псевдонимом для `IEmitDefault<ICustomer>`

### Классы моделей:

Каталог товаров

**Конструктор** отсутствует

**Методы класса:**

#### Класс Customer

Информация о покупателе

**Конструктор** отсутствует

**Методы класса:**

`setPayment(payment: TPayment): void`,
`setEmail(email: string): void`,
`setPhone(phone: string): void`,
`setAddress(address: string): void` - принимают строковые значения

#### Класс Cart

Корзина покупок

**Конструктор** отсутствует

`removeItem(item: IProduct): void` - удаляет товар из корзины

#### Класс AppApi

**Методы класса:**
`getProducts(): Promise<IProductsResponse>` - не принимает аргументы, возращает

`postOrder(order: IOrderData): Promise<IOrderResponse>` - принимает объект

events: EventEmitter,
basketTemplate: HTMLTemplateElement,
itemTemplate: HTMLTemplateElement`

**Методы класса:**

`updateBasket(products: IProduct | IProduct[], totalPrice: number): void` - обновляет отображение модального окна с корзиной

`showModal(): void` - открывает модальное окно с корзиной

#### Класс CardPreview

Класс отвечает за отображение модального окна с подробной информацией о товаре. Наследуется от `ModalView` и реализует визуальное представление карточки товара с возможностью добавления или удаления из корзины.

**Конструктор** принимает следующие параметры:

`container: HTMLElement, events: EventEmitter, itemPreviewTemplate: HTMLTemplateElement`

**Методы класса:**

`setActiveCard(product: IProduct, isInBasket: boolean): void` - устанавливает данные товара в модальном окне и настраивает кнопку действия в зависимости от наличия товара в корзине

`showModal(): void` - открывает модальное окно с карточкой товара

#### Класс CatalogView

Класс отвечает за отображение каталога товаров на главной странице. Наследуется от `Component` и реализует рендеринг карточек товаров в виде списка.

**Конструктор** принимает следующие параметры:

`container: HTMLElement, events: EventEmitter, cardTemplate: HTMLTemplateElement`

**Методы класса:**

`addCards(products: IProduct | IProduct[]): void` - добавляет карточки товаров в каталог. Принимает один товар или массив товаров, для каждого создает DOM-элемент на основе шаблона и добавляет обработчик клика, генерирующий событие `catalogCard:click`

#### Класс HeaderView

Класс отвечает за отображение шапки сайта. Наследуется от `Component` и реализует отображение счетчика товаров в корзине и обработку клика по иконке корзины.

**Конструктор** принимает следующие параметры:

`container: HTMLElement, events: EventEmitter`

**Методы класса:**

`setBasketCounter(num: number): void` - обновляет счетчик товаров в корзине на переданное число

#### Класс ModalView

Базовый класс для всех модальных окон. Наследуется от `Component` и реализует базовую логику открытия, закрытия и управления содержимым модального окна.

- `order:submit` - генерируется при отправке формы заказа

### Классы представления

#### Класс ApiPresenter

Класс-презентер, отвечающий за взаимодействие с сервером. Обрабатывает получение товаров и отправку заказов, координируя работу API, корзины и данных покупателя.

**Конструктор** принимает следующие параметры:

`api: AppApi, events: EventEmitter, basket: Basket, customer: Customer`

**Методы класса:**

`getItems(): Promise<void>` - запрашивает список товаров с сервера и генерирует событие `appApi:getItems` с полученными данными

`postOrder(): Promise<void>` - собирает данные заказа из корзины и информации о покупателе, отправляет на сервер и генерирует событие `appApi:orderPosted` с ответом сервера при успешном выполнении

**События, генерируемые классом:**
- `appApi:getItems` - генерируется после получения списка товаров, передает ответ сервера (тип `IFetchItemsResponse`)
- `appApi:orderPosted` - генерируется после успешной отправки заказа, передает ответ сервера (тип `IEmitDefault<IPostOrderResponse>`)

#### Класс BasketPresenter

Класс-презентер, отвечающий за управление корзиной покупок. Координирует работу модели корзины и представления корзины, обрабатывает отображение и удаление товаров.

**Конструктор** принимает следующие параметры:

`basket: Basket, view: BasketModalView`

**Методы класса:**

`showBasket(): void` - отображает текущее содержимое корзины и открывает модальное окно

`updateBasket(): void` - обновляет отображение корзины без открытия модального окна

`deleteItem(res: TEmitProduct): void` - удаляет товар из корзины по переданным данным и обновляет отображение

#### Класс CardPreviewPresenter

Класс-презентер, отвечающий за управление предпросмотром товара. Координирует работу модели товаров, корзины и представления карточки товара, обрабатывает открытие карточки и добавление/удаление товара из корзины.

**Конструктор** принимает следующие параметры:

`model: Products, basket: Basket, view: CardPreview`

**Методы класса:**

`openPreview(res: TEmitProduct): void` - открывает модальное окно с карточкой товара, устанавливает выбранный товар в модели и проверяет наличие товара в корзине

`onActionBtnClick(res: TEmitProduct): void` - обрабатывает нажатие на кнопку действия (добавить/удалить), переключает состояние кнопки и обновляет корзину

`onClose(): void` - очищает выбранный товар в модели при закрытии модального окна

#### Класс CatalogPresenter

Класс-презентер, отвечающий за управление каталогом товаров. Координирует работу модели товаров и представления каталога, обрабатывает отображение списка товаров.

**Конструктор** принимает следующие параметры:

`model: Products, view: CatalogView`

**Методы класса:**

`renderCatalog(items: IProduct[]): void` - сохраняет полученные товары в модели и отображает их в каталоге

#### Класс ContactsPresenter

Класс-презентер, отвечающий за управление формой контактов. Координирует работу модели покупателя и представления контактов, обрабатывает ввод данных и валидацию.

**Конструктор** принимает следующие параметры:

`customer: Customer, view: ContactsView`

**Методы класса:**

`showModal(): void` - открывает модальное окно с формой контактов

`onInput(res: TEmitUserData): void` - обрабатывает ввод данных в форму, обновляет модель покупателя, выполняет валидацию и отображает ошибки

#### Класс HeaderPresenter

Класс-презентер, отвечающий за управление шапкой сайта. Координирует работу корзины и представления шапки, обновляет счетчик товаров.

**Конструктор** принимает следующие параметры:

`basket: Basket, view: HeaderView`

**Методы класса:**

`updateBasketCounter(): void` - получает количество товаров из корзины и обновляет счетчик в шапке

#### Класс OrderPresenter

Класс-презентер, отвечающий за управление формой заказа. Координирует работу модели покупателя и представления заказа, обрабатывает выбор способа оплаты и ввод адреса доставки с валидацией.

**Конструктор** принимает следующие параметры:

`customer: Customer, view: OrderModalView`

**Методы класса:**

`showModal(): void` - открывает модальное окно с формой заказа

`onInput(res: TEmitPartialUserData): void` - обрабатывает ввод данных в форму. При выборе способа оплаты переключает состояние (повторный клик отменяет выбор), обновляет модель покупателя, выполняет валидацию и отображает ошибки

#### Класс SuccessPresenter

Класс-презентер, отвечающий за управление окном успешного оформления заказа. Координирует работу корзины и представления успешного заказа, отображает информацию о списании и очищает корзину.

**Конструктор** принимает следующие параметры:

`basket: Basket, events: EventEmitter, view: SuccessView`

**Методы класса:**

`showSuccessWindow(res: IEmitDefault<IPostOrderResponse>): void` - отображает окно с информацией о списанной сумме, очищает корзину и генерирует событие `basket:clear`
